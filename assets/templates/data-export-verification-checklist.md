# Data Export Verification Checklist — User Side

> **STARTING-POINT NOTICE.** Substantive review by qualified counsel is required.

Use on receipt of an Article 4(1) data export from a Data Holder, or an Article 5(1) export to a third party on the User's behalf. The checklist verifies that the data holder has met the format, quality, latency, and completeness requirements.

## 1. Format

- [ ] Data is in a comprehensive, structured, commonly used and machine-readable format. (Examples: JSON, XML, CSV; not licensing-restricted formats.)
- [ ] Format is documented (schema reference, column definitions).
- [ ] Format matches the format described in pre-contract notice and in any DPS register (Art. 26(b)).

## 2. Completeness

- [ ] All categories of product data described in the pre-contract notice (Art. 3(2)(a) / 3(3)(a)–(b)) are present.
- [ ] Metadata necessary to interpret the data is included (Art. 4(1)).
- [ ] Volumes are consistent with the volume estimates in the pre-contract notice.
- [ ] Where related service data is in scope, related-service-data categories are present.

## 3. Quality

- [ ] Data appears as accurate, complete, reliable, relevant and up-to-date as it would be available to the data holder. (Recital 30; FAQ Q22a.)
- [ ] No evidence of redaction beyond the data holder's identified trade-secret-protected categories.
- [ ] Where derived / inferred data was excluded, the exclusion is consistent with FAQ Q4 / Q5.

## 4. Timeliness

- [ ] Response was without undue delay relative to the request date.
- [ ] If the data holder asserted a delay justification (security / technical / legal), the justification is proportionate.

## 5. Channel

- [ ] Channel is secure (encryption in transit; authenticated endpoint).
- [ ] Free of charge to the user (Art. 4(1)).

## 6. Identification of trade-secret-protected categories

- [ ] Trade-secret-protected data is identified in the metadata (Art. 4(6)).
- [ ] If withheld, the data holder has provided a written substantiation under Art. 4(7) or 4(8).
- [ ] If withheld, competent-authority notification confirmed.

## 7. Personal data

- [ ] Personal data of the User is included where applicable.
- [ ] Personal data of third parties (e.g., employees) is handled consistent with GDPR — anonymised, redacted, or accompanied by a separate Article 6 GDPR legal basis where the User is not the data subject (Recital 7; FAQ Q25a).

## 8. Third-party transfer (if applicable)

- [ ] If the export was an Article 5 transfer to a third party, the receipt confirmation from the third party is on file.
- [ ] Third party's confirmation that it is not a designated gatekeeper (Art. 5(3)).
- [ ] Trade-secret confidentiality measures with the third party are in place (Art. 5(9)).

## 9. Use restrictions (Article 4(10))

The User confirms it will not:

- [ ] use the data to develop a connected product that competes with the connected product from which the data originate;
- [ ] share the data with a third party with that intent; or
- [ ] derive insights about the manufacturer's or data holder's economic situation, assets or production methods.
